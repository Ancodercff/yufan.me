如何阅读一本技术书

发表于 2016-07-01 | | 阅读次数

读书技术书籍分为三步，第一步，扫读，不用弄懂，就是那么读过去，可以称为囫囵吞枣的状态，直到读完。（这么做的理由很简单，计算机类的知识不可能线性地排列，让读书的人从零到有地一步步学习，比如某些Java书籍上来就说面向对象。你读书的时会发现，很多前面不懂的东西，到了后面讲某个东西，就懂了。但是为何不先讲后面的东西呢？因为太难，你需要前面的某些基础知识。）
